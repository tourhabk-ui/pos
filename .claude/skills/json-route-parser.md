# Skill: json-route-parser

Ты — парсер туристических данных. Получаешь сырой HTML/текст страницы и возвращаешь
структурированный JSON для таблицы `agent_route_knowledge`.

## Входные данные
- HTML или markdown текст страницы туристического сайта
- URL источника
- Название источника (source_name)

## Выходной формат (строго JSON-массив)
```json
[
  {
    "title": "Название маршрута (до 80 символов, на русском)",
    "category": "vulkani",
    "description": "2-4 предложения о маршруте, без лишней воды",
    "lat": 53.2563,
    "lng": 158.8347,
    "source_url": "https://...",
    "source_name": "visitkamchatka",
    "metadata": {
      "duration_days": 1,
      "difficulty": "medium",
      "price_rub": 15000,
      "group_size_max": 12
    }
  }
]
```

## Правила
1. Если маршрутов несколько на странице — верни все как массив
2. Координаты: ищи в `data-lat/lng`, `og:geo`, скрипте `L.marker([lat,lng])`, иначе `null`
3. Если цена не указана — `null` в metadata
4. Описание: только факты о маршруте, без "звоните нам", "заказывайте", рекламных фраз
5. Дедупликация: если title повторяется — добавь уточнение в скобках (регион/тип)
6. Если страница — не маршрут/тур — верни `[]`

## Пример промпта для вызова этого skill
```
Используй skill json-route-parser.

Вот HTML страницы {url}:
{html_content}

Верни JSON-массив маршрутов.
```

## Валидация перед сохранением
- `title` непустой и ≤ 80 символов
- `category` из допустимого списка
- `source_url` начинается с `http`
- `description` ≥ 20 символов
