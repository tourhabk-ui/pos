# Структура категорий: 3 оси классификации

> ПРАВИЛО: Одна точка на карте = много товаров.
> Товар = ГДЕ + ЧТО + КАК + КТО + ЦЕНА + КОГДА

---

## Пример: Авачинский вулкан

```
АВАЧИНСКИЙ ВУЛКАН (location_type: volcano) — 1 точка на карте
│
│  ЧТО делать          КАК добраться       КТО продаёт        ЦЕНА      КОГДА
│  ─────────────────   ──────────────────   ────────────────   ────────   ──────────
├── восхождение        пешком сам           (бесплатно)        0₽         июнь-сент
├── восхождение        пешком с гидом       Камчатинтур        5 000₽     июнь-сент
├── восхождение        джип + пешком        ВулканГид          8 000₽     июнь-сент
├── восхождение        вахтовка + пешком    ТопКам             6 000₽     июнь-сент
├── катание            снегоход             СнегоТур           12 000₽    дек-март
├── облёт              вертолёт             КамчатХели         35 000₽    круглый год
├── восхождение        вертолёт + пешком    КамчатХели         45 000₽    июнь-сент
└── фрирайд            вертолёт + лыжи      HeliSki            80 000₽    март-май
```

## Как это ложится на данные

### Уровень 1: agent_route_knowledge (МЕСТО)
```
id: uuid
title: "Авачинский вулкан"
location_type: "volcano"        ← НОВОЕ
lat: 53.255
lng: 158.836
description: "Действующий вулкан..."
```

### Уровень 2: tours (ТОВАР)
```
id: uuid
route_id: → agent_route_knowledge.id
operator_id: → partners.id
activity_type: "trekking"        ← ЧТО
transport_type: "on_foot"        ← КАК
title: "Восхождение на Авачинский"
price: 5000
season: [6, 7, 8, 9]
duration_hours: 12
difficulty: "medium"
max_group: 15
```

### Связь
```
agent_route_knowledge (1 запись = 1 место)
  └── tours (N записей = N товаров от разных операторов)
       ├── tour 1: trekking + on_foot + Камчатинтур + 5000₽
       ├── tour 2: trekking + jeep + ВулканГид + 8000₽
       ├── tour 3: ride + snowmobile + СнегоТур + 12000₽
       └── tour 4: sightseeing + helicopter + КамчатХели + 35000₽
```

## Правила для UI

### Карта (/map)
- Точка = МЕСТО (location_type)
- Иконка маркера = по location_type
- Клик на точку → попап с названием + кол-во доступных туров
- Далее → страница места с полным списком товаров

### Каталог (/routes)
- Фильтр 1: ЧТО хочешь делать? (activity_type)
- Фильтр 2: КАК добраться? (transport_type)
- Фильтр 3: ГДЕ? (location_type) — дополнительный
- Фильтр 4: бюджет, даты, сезон

### Страница места (/routes/[id])
- Заголовок = название места + location_type
- Описание + координаты + фото
- ТАБЛИЦА ТОВАРОВ: activity + transport + operator + price + date + [кнопка]

## Геометрия мест — point, polyline, polygon

Не все МЕСТА на карте — это точки. Некоторые объекты — линии или зоны.

### Типы геометрии

| Тип | Пример | На карте | Хранение |
|-----|--------|----------|----------|
| `point` | Авачинский вулкан, горячий источник | Маркер | `lat` + `lng` |
| `polyline` | Участок реки (рыболовная зона), береговая линия | Цветная линия + маркер в центре | `payload.geometry` — массив точек |
| `polygon` | Кальдера, залив, заповедник | Закрашенный многоугольник | `payload.geometry` — замкнутый контур |

### Правило для UI

- `point`: маркер в `[lat, lng]`
- `polyline`: цветная линия по координатам + маркер в центре линии (центроид) для клика / попапа
- `polygon`: закрашенная區域 + маркер в центре для попапа

### Хранение geometry в payload

```json
{
  "geometry": {
    "type": "polyline",
    "coordinates": [
      [55.4461, 159.6731],
      [55.4433, 159.5003],
      [55.4422, 159.5022],
      [55.4444, 159.6742]
    ]
  }
}
```

Поле `lat`/`lng` = центроид (для списков, поиска, маркера). `payload.geometry` = полная форма (для рисования).

## Правила именования мест (place naming)

### Когда имя = географическое название

Стандартный случай. Место существует независимо от оператора.

```
"Авачинский вулкан"        — вулкан, существует сам по себе
"Долина гейзеров"          — природный объект
"Курильское озеро"         — озеро
"Халактырский пляж"        — пляж
```

### Когда имя = бренд оператора

Место является **лицензированной территорией** одного оператора. Конкуренты туда не ходят.
Пример: рыболовный участок с лицензией.

```
"Рыболовная база «Камчатская рыбалка»"
  — это НЕ просто "река Камчатка"
  — это конкретный участок №1182, лицензия одного оператора
  — назвать безлично нельзя — другие операторы на этот участок не ходят
  — location_type: river, activity_type: fishing
  — geometry: polyline (участок реки ~21 км)
```

### Правило выбора

1. Если на этот объект могут вести туры 2+ разных оператора → **географическое название**
2. Если место = эксклюзивная территория одного оператора (лицензия, база) → **бренд оператора**
